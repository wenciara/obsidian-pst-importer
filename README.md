# PST Import

Import Outlook PST email archives into Obsidian as Markdown files with metadata, attachments, and wikilinks.

## Key Use Cases

### Use case 1 — Outlook Auto-Archive (ongoing sync)

Outlook's **Auto-Archive** feature periodically moves old emails into a local `.pst` file. PST Import can keep that archive continuously synced into Obsidian.

**Workflow:**

1. Configure Outlook Auto-Archive: File → Options → Advanced → AutoArchive Settings
   - Set it to run automatically and archive to a fixed path (e.g. `C:\Users\you\Documents\OutlookArchive.pst`)
2. In PST Import settings, click **Add Sync Profile** and point it to that `.pst` file
3. Run **"Sync configured PST (incremental)"** from the command palette periodically
   - Only emails added since last sync are imported — no duplicates

You can add **multiple sync profiles** (e.g. one for work archive, one for a project-specific PST).

### Use case 2 — One-time import of any PST mailbox

You can import any PST file as a standalone archive — a backup export from an old account, a handoff from a colleague, a completed project mailbox, etc.

Each import creates its own folder in your vault. You name it at import time.

---

## Features

- 📧 Export all emails from PST files as Markdown
- 🔄 **Incremental sync** — only imports new emails since last sync, no duplicates
- 📁 Preserve PST folder structure (Inbox, project subfolders, etc.)
- 🖼️ Embedded images auto-convert to Obsidian `![[wikilink]]` format
- 📎 Attachments saved to `attachments/` subfolder per email folder
- 📋 YAML frontmatter with full metadata (sender, recipients, date, categories)
- 🚀 Pure JavaScript engine — no Outlook installation or external tools required
- 📂 Pre-import folder selection, discard all-empty mailboxes

## Usage Guide

### Scenario A: Importing a one-time PST archive

1. Click the **PST icon** in the ribbon, or run **"Import PST file..."** from the command palette
2. Select a `.pst` file
3. In the folder selection dialog, uncheck any folders you don't need (e.g. Sent Items, Drafts), then click **Import**
4. The import runs and shows a progress window. Large PST files (1–5 GB) typically take 3–10 minutes.

> **Tip:** You can import the same PST again later into a different folder name if needed. Turn on "Overwrite Existing" only if you want to refresh notes you may have edited.

---

### Scenario B: Incremental sync with Outlook Auto-Archive

1. Go to **Settings → PST Import → Add Sync Profile**
2. Select your Auto-Archive `.pst` file — PST Import scans all email folders automatically
3. Optionally deselect folders you never want synced (you can edit this later)
4. Run **"Sync configured PST (incremental)"** from the command palette
   - First run: imports all existing emails
   - Subsequent runs: imports only new emails added since last sync

**Naming your sync output folder:**

When setting up the sync profile, the output folder defaults to the PST filename. You can rename it in the profile. Recommended patterns:

| Archive type | Output folder name |
|---|---|
| Main Outlook auto-archive | `Email` |
| Work account archive | `Work Email` |
| Project-specific PST | `ProjectName / Email` |

---

### Folder selection rules

- Only folders that contain emails are shown — empty folders are automatically hidden
- System folders (Contacts, Calendar, Tasks, Sync Issues, Conversation History, etc.) are excluded by default
- Subfolders appear as indented paths, e.g. `Inbox/ProjectA`

## Large PST Files

PST Import uses a streaming reader and processes emails one at a time, so there is **no hard file-size limit**. Modern Unicode PST files (generated by Outlook 2003 and later) of 3 GB, 5 GB, or more can be imported — but there are practical considerations:

| File size | Expected behavior |
|-----------|------------------|
| < 1 GB | Fast, typically completes in under 5 minutes |
| 1–5 GB | Works, but may take 15–45 minutes; Obsidian will appear busy |
| 5–20 GB | Will complete, but can take hours; best to import overnight |
| > 20 GB | Possible, but risk of memory exhaustion increases if emails carry many large attachments |

> **Note:** The 2 GB warning sometimes seen online applies only to the older ANSI PST format (Outlook 97–2002). Outlook has generated Unicode PST files by default since Outlook 2003, and those have no built-in size limit.

### Recommended approach for very large archives

Instead of importing the entire PST at once, **use Outlook to export individual folders as separate, smaller PST files**, then import each one:

1. In Outlook, right-click a folder → **Export** → **Export to a File** → **Outlook Data File (.pst)**
2. Select only the folder(s) you want (e.g. one project folder or one year's worth of mail)
3. Save as a new `.pst` file with a descriptive name (e.g. `ProjectAlpha_2023.pst`)
4. Import that smaller file with PST Import

Splitting by **project**, **year**, or **sender domain** keeps each imported vault folder focused and fast to process.

### From Obsidian Community Plugins

1. Open Obsidian → Settings → Community Plugins → Browse
2. Search for **"PST Import"**
3. Click Install → Enable

### Install with BRAT

1. Install [BRAT](https://obsidian.md/plugins?id=obsidian42-brat)
2. Add repository: `wenciara/obsidian-pst-importer`
3. Enable PST Import

### Manual Installation

1. Download `main.js`, `manifest.json`, and `styles.css` from [Releases](https://github.com/wenciara/obsidian-pst-importer/releases)
2. Place files in `.obsidian/plugins/pst-import/`
3. Enable in Obsidian settings

## Settings

| Setting | Description | Default |
|---------|-------------|---------|
| Output Base Folder | Default vault folder for imports | *(empty)* |
| Mirror Folder Structure | Create subdirectories matching PST folders | Enabled |
| Overwrite Existing Files | Replace existing notes on re-import | Disabled |
| Include YAML Frontmatter | Add metadata header to each email | Enabled |
| **Sync Profiles** | Configure PST paths for incremental sync | *(none)* |

## Output Structure

```
vault/
└── Email Archive/          ← folder name you choose at import time
    ├── Inbox/
    │   ├── 2024-03-15_001_Alice_Project kickoff.md
    │   └── attachments/
    │       ├── proposal.pdf
    │       └── logo.png
    ├── Inbox/ProjectA/
    │   └── 2024-04-02_003_Bob_Re - timeline.md
    └── Sent Items/
        └── ...
```

Each email filename follows the pattern: `date_index_sender_subject.md`
